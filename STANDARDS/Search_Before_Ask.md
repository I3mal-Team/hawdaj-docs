# Search_Before_Ask — ابحث قبل أن تسأل (إلزامية)

> **سياسة حاكمة.** قبل طرح أي سؤال توضيحي على المستخدم، **ابحث في قاعدة المعرفة عن حوادث مشابهة**. لا تسأل عمّا هو موثّق أصلًا.
> يُرجَع لهذا المستند من `CLAUDE.md` و`WORKFLOWS/*` (خاصة BUG_FIX). لا يُكرَّر — يُشار إليه.

---

## القاعدة
> **Before asking the user any clarification, search the Knowledge Base for similar incidents.**

المستخدم غالبًا يعطي وصفًا مختصرًا/غامضًا. مسؤوليتك أن تستنتج من التوثيق أولًا — السؤال آخر ملاذ، لا أوّل خطوة.

## متى تُطبَّق
عند أي طلب **غامض** أو **عطل غير واضح** (bug، سلوك خاطئ، رسالة خطأ، "مش شغّال") — قبل استخدام أي أداة سؤال.

## التدفّق الإلزامي
```
مشكلة/طلب غير واضح
        ↓
[1] ابحث في قاعدة المعرفة (بالترتيب):
    • FEATURES/NN_*.md → قسم Common Bugs
    • FEATURES/NN_*.md → قسم Debug Guide
    • FEATURES/NN_*.md → قسم Known Incidents / Edge Cases
    • KNOWN_ISSUES.md  (K1..)
    • DECISIONS/TECH_DEBT.md (D1..)
    • CHANGE_LOG.md (هل عُولج سابقًا؟)
    • SEARCH_INDEX.md (كلمة مفتاحية → ميزة) · API_INDEX / DATABASE_INDEX عند الحاجة
        ↓
[2] وُجدت حالة مشابهة؟
        ↓ نعم                          ↓ لا
   اعرضها مع نسبة التشابه          اسأل المستخدم سؤالًا واحدًا فقط
   (الملف + السطر + الحل الموثّق)   (محدّدًا، لا مفتوحًا)
        ↓
   أكمِّل التشخيص/الخطة بناءً عليها
```

## نسبة التشابه (كيف تُقدَّر وتُعرَض)
- **مرتفع (≈80–100%):** نفس العَرَض + نفس المكوّن/الميزة → اعرض الحالة كسبب مرجّح مباشرة.
- **متوسط (≈50–79%):** عَرَض مشابه، مكوّن مختلف أو ظروف مختلفة → اعرضها كفرضية وتحقّق.
- **منخفض (<50%):** تلامس جزئي → اذكرها كمرجع، ثم اسأل سؤالًا واحدًا.
- **صيغة العرض:** `تشابه ~NN% — [[KNOWN_ISSUES#Kx]] / FEATURES/NN §Common Bugs — <العَرَض المطابق> — <الحل الموثّق>`.

## قاعدة السؤال الواحد
- لو لم تجد أي حالة: **سؤال واحد محدّد فقط** (أي شاشة؟ رسالة الخطأ الحرفية؟ status code؟) — لا قائمة أسئلة، لا سؤال مفتوح.
- لو وجدت حالة مرتفعة التشابه: **لا تسأل** — اعرض الحل واطلب تأكيدًا فقط إن لزم.

## أمثلة (من حوادث موثّقة)
- "المشرف ما يدخل / ما تجيه كلمة السر" → قبل السؤال: `KNOWN_ISSUES#K4` (بريد queued + catch صامت) + `#K1` (SMTP) → تشابه مرتفع → اعرض مباشرة.
- "التقييم بصفر مقبول" → `FEATURES/14 §Common Bugs` + `TECH_DEBT#D4` → تشابه مرتفع.
- "trips/save-trip-to-email يرجّع 500" → `KNOWN_ISSUES#K1` (SMTP 587) → تشابه مرتفع.
