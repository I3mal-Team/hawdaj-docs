# REGRESSION_CHECKLIST — قائمة الانحدار

> يرجع لها كل تدفّق في الخطوة 7/10. لكل ميزة قسم **Regression Checklist** خاص بمستندها — هذه القائمة عامّة + المسارات الحرجة المشتركة.

## كيف تستخدمها
1. من مستند الميزة المتأثّرة، انسخ قسم **Regression Checklist**.
2. أضِف المسارات المشتركة أدناه المتأثّرة بالتغيير.
3. من `FEATURE_GRAPH.md`: أضِف الميزات المعتمِدة على المتغيّر.
4. حدّد نوع كل اختبار (unit/integration/widget/smoke).

## مسارات حرجة مشتركة (تحقّق دائمًا لو لُمِست)
- [ ] **المصادقة:** login/register/logout · social (`/social/callback` ويب) · token يُحفظ/يُرفق · 401 يتصرّف صحيح.
- [ ] **الإقلاع (موبايل):** ForceUpdate gate → Splash → onboarding/home · locale محمّل بلا وميض.
- [ ] **الشبكة:** envelope يُفكّ صحيح · خطأ يُعرض بلطف · offline → شاشة مناسبة.
- [ ] **التعريب:** تبديل ar/en/ru/zh يغيّر النصوص · RTL للعربية · لا مفاتيح مكشوفة.
- [ ] **القوائم:** ترقيم (صفحة 2 تحمّل) · حالة فارغة · حالة خطأ.
- [ ] **المحتوى:** list + detail لـPlaces/Stores/Zad/Events/Swalef/Guides يفتح بلا crash.
- [ ] **التفاعل:** favorite/save toggle · rate submit · flags (is_favorite/is_saved) صحيحة.
- [ ] **الوسائط:** رفع صورة (حجم/نوع) · عرض صورة · fallback عند الكسر.
- [ ] **UGC:** إنشاء (landmark/property/story) → status صحيح · يظهر في "my *".
- [ ] **الاتساق:** لو عقد API تغيّر → الموبايل + الويب يعملان.

## بوابة الأمان (لو التغيير أمني)
- [ ] المسار الشرعي يعمل. [ ] الطلب بلا auth يُرفض. [ ] القيم خارج المدى تُرفض. [ ] لا تسريب PII في الاستجابة.

## قاعدة الحدّ الأدنى لأي bug fix
- [ ] الحالة اللي كانت تعطل صارت تعمل.
- [ ] الحالة العكسية سليمة.
- [ ] حالة حافّة واحدة.
- [ ] ميزة مرتبطة واحدة (من FEATURE_GRAPH) لم تنكسر.
