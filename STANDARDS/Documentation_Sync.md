# Documentation_Sync — سياسة مزامنة التوثيق (إلزامية)

> **سياسة حاكمة.** `hawdaj-docs` هو **مصدر الحقيقة الوحيد (Single Source of Truth)** لكامل نظام Hawdaj (موبايل + باك + ويب). **التوثيق يجب ألّا يصبح أقدم من التنفيذ أبدًا.**
> يُرجَع لهذا المستند من `CLAUDE.md`، `WORKFLOWS/*`، و`STANDARDS/Completion_Gate.md`. لا يُكرَّر — يُشار إليه.

---

## 1. المبدأ
- الكود يعيش في ريبوهات منفصلة (`hawdaj/Untitled`, `hawdaj-api`, `hawdaj-frontend`)، والتوثيق في ريبو مستقل (`/Users/mac/hawdaj-docs`).
- لأنهما **منفصلان**، أي تغيير كود قد يجعل التوثيق قديمًا بصمت. لذلك المزامنة **إلزامية وجزء من التنفيذ** — ليست اختيارية ولا لاحقة.
- **التوثيق جزء من التنفيذ. المهمة لا تكتمل قبل مزامنته ورفعه.**

## 2. متى تُطبَّق
عند أي تغيير كود في **Flutter / Laravel / Angular**، يجب على المهندس (بشري أو AI) أن يقرّر: **هل يؤثّر التغيير على قاعدة المعرفة المشتركة؟**

## 3. تدفّق القرار الإلزامي
```
Code Change
   ↓
هل يؤثّر على التوثيق المشترك؟
   ↓                       ↓
  NO                      YES
   ↓                       ↓
Continue            Update hawdaj-docs
                          ↓
                  Commit documentation changes
                          ↓
                  Push documentation repository
                          ↓
                     Task Complete ✅
```
"يؤثّر" = يمسّ: سلوك/عقد ميزة · endpoint (API) · جدول/migration (DB) · تبعية بين ميزات · إصلاح bug · حادثة تشغيلية · دَين تقني · قرار معماري.

## 4. التحذير الإلزامي (إذا كان التغيير يؤثّر على التوثيق)
عند اكتشاف أثر على قاعدة المعرفة، **توقّف وأبلغ حرفيًا:**

> ⚠️ تم اكتشاف أن هذا التعديل يؤثر على قاعدة المعرفة المشتركة.
> يجب تحديث الملفات المناسبة داخل مستودع hawdaj-docs.
> لا تعتبر المهمة مكتملة حتى يتم تحديث ورفع التوثيق.

## 5. ما الذي يُحدَّث (حسب نوع التغيير)
| نوع التغيير | حدّث في hawdaj-docs |
|-------------|---------------------|
| أي تغيير | `CHANGE_LOG.md` (سطر) · `PROJECT_PROGRESS.md` (سرد) |
| سلوك/منطق ميزة | `FEATURES/NN_*.md` (+ قسم "Web Frontend" لو الويب) |
| endpoint/عقد API | `API_INDEX.md` + قسم API بمستند الميزة |
| جدول/migration/فهرس | `DATABASE_INDEX.md` |
| تبعية بين ميزات | `FEATURE_GRAPH.md` |
| حادثة تشغيلية/بيئية | `KNOWN_ISSUES.md` |
| دَين جديد/مُسدَّد | `DECISIONS/TECH_DEBT.md` |
| قرار معماري | `DECISIONS/ARCHITECTURE_DECISIONS.md` |
| اختلاف ويب↔موبايل | `WEB_MOBILE_COMPARISON.md` |

## 6. بوابة الإتمام (المهمة INCOMPLETE إذا…)
المهمة **❌ INCOMPLETE** تلقائيًا لو:
- التوثيق قديم (لا يعكس الكود).
- `CHANGE_LOG` لم يُحدَّث.
- `PROJECT_PROGRESS` لم يُحدَّث.
- مستندات الميزات المتأثّرة لم تُحدَّث.
- فهارس API/DATABASE/GRAPH قديمة.
- `KNOWN_ISSUES` أو `TECH_DEBT` كان يجب تحديثهما ولم يُحدَّثا.
- **تغييرات التوثيق لم تُودَع (commit) وتُرفَع (push) إلى `hawdaj-docs`.**

## 7. الرفع إلزامي (Commit + Push)
تحديث الملفات محليًا **لا يكفي**. الخطوة الأخيرة في التدفّق: `commit` + `push` لريبو `hawdaj-docs` — وإلا يبقى المطوّرون/الوكلاء الآخرون على توثيق قديم. المهمة تكتمل فقط بعد الدفع.
