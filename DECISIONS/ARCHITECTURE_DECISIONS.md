> الصيغة: القرار · السياق · النتيجة · البديل المرفوض (إن وُجد).

## قرارات قائمة (Observed — كما هو الكود اليوم)

**A1 — Flutter: Cubit بدل Bloc/Riverpod.** الحالة عبر `flutter_bloc` (Cubit) + `get_it` DI. النتيجة: نمط موحّد بسيط لكل feature. أي feature جديدة تتبعه.

**A2 — Either<Failure,T> (dartz) لمعالجة الأخطاء.** كل repo يرجّع `Either`. النتيجة: أخطاء صريحة بلا exceptions متسرّبة. إلزامي في كل طبقة data.

**A3 — Feature-First + Clean رفيعة.** `lib/features/<f>/{data,presentation}`. لا طبقة domain كاملة (مبسّطة). النتيجة: عزل الميزات، سهولة التوثيق.

**A4 — go_router للتنقّل.** مسارات مركزية. النتيجة: deep-link + تنظيم.

**A5 — Backend: فصل حارسين.** `auth('api')` (Sanctum) للعملاء (موبايل+ويب) · `auth:web` + Spatie للوحة. النتيجة: فصل نظيف. **لكن** الصلاحيات guard=web فقط → الـAPI بلا authz دقيق (دَين D6).

**A6 — Astrotomic Translatable لكل المحتوى.** `{model}_translations` + مصدر عربي. النتيجة: 4 لغات. البديل المرفوض ضمنيًا: أعمدة مترجمة مباشرة.

**A7 — كشط Google Translate عبر Observers (لا API رسمي).** ترجمة تلقائية متزامنة عند الحفظ. النتيجة: تعبئة تلقائية **لكن هشّة** (دَين D18). قرار يُعاد النظر فيه (queue + رسمي).

**A8 — Trips: تعايش v1 + v2.** v2 (enhanced) هو المعتمَد؛ v1 legacy باقٍ. النتيجة: دَين انتقالي. الجديد يبني على v2 فقط.

**A9 — نظام Media ثلاثي (Spatie + galleries + legacy).** تاريخي. النتيجة: تعقيد + دَين D28. القرار المستقبلي: التوحيد على Spatie.

**A10 — Force Update عبر Firebase Firestore (لا Laravel).** بوابة الإصدار خارج الباك. النتيجة: fail-open عند الانقطاع (دَين D32). قرار يُعاد النظر (نقل/عكس للباك).

**A11 — Web: Angular SSR + خدمات RxJS (لا NgRx).** SEO أولًا. النتيجة: FCP سريع + meta/hreflang/canonical. حالة عبر BehaviorSubjects.

**A12 — الرئيسية: عقدان مختلفان.** موبايل نداء واحد مجمّع `/home`؛ ويب 8+ نداءات دقيقة. النتيجة: مساران للصيانة (دَين ضمني). قرار مفتوح: توحيد أو توثيق العقد المقصود.

## القاعدة الذهبية (قرار حاكم)
**أي تغيير يمسّ عقد API يبدأ من Backend، ثم يُحدَّث الموبايل والويب على العقد الجديد، ثم يُوثَّق.** سبب: عميلان يتشاركان نفس الـAPI؛ تغيير أحادي الجانب يكسر الآخر.

### ADR-01 — البريد غير قاتل عالميًا عبر `SafeMailTransport` (2026-07-08)
**القرار:** تغليف transport البريد الحقيقي بـ`App\Mail\SafeMailTransport` decorator يبتلع أي استثناء أثناء الاتصال/الإرسال ويسجّله بدل رميه، مسجّل عالميًا في `AppServiceProvider::boot()` — يغطّي `Mail::send` وقناة mail للإشعارات (queued وغير queued). · **السياق:** خادم الإنتاج يحجب SMTP الصادر (587) فيرمي `Swift_TransportException` يكسر طلبات HTTP وjobs (راجع [[KNOWN_ISSUES#K1]] · [[TECH_DEBT#D45]]). · **النتيجة:** فشل البريد لم يعد يكسر أي تدفّق؛ لكن **صار صامتًا عالميًا (لوق فقط)** — لا تسليم ولا خطأ ظاهر، ولا يعالج الجذر (البورت/التزامن). مقايضة: مرونة مقابل رؤية. · **البديل المرفوض:** `try/catch` نقطي في كل مُرسِل (متكرر، سهل النسيان) أو إصلاح البنية (queue + مزوّد API على 443) — مؤجَّل.

### ADR-02 — ترتيب home بالموقع خارج الـcache + إبطال بالنسخة (2026-07-08)
**القرار:** الطلبات ذات الموقع تحسب home طازجًا بالإحداثيات الدقيقة وتتخطّى الـcache؛ الطلبات بلا موقع تُخزَّن 10د بمفتاح يحمل `home_cache_version` يُرفَع من `PlaceObserver::saved()/deleted()` على Place/Event/Store/ZadElgadel. · **السياق:** cache عام سابق سبّب تلوّثًا بالموقع (ترتيب مستخدم يسرّب لآخر) وظهورًا بطيئًا للمحتوى (حتى TTL) بلا أي invalidation. · **النتيجة:** ترتيب حيّ دقيق + ظهور فوري للمحتوى الجديد؛ مقابل حساب أثقل (~40–50 استعلام) لكل طلب ذي موقع بلا تخفيف cache. · **البديل المرفوض:** bucket ~1كم في المفتاح (يجمّد الترتيب ويظل يلوّث) — أُزيل. مقترح لاحق: bucket geohash قصير العمر لطلبات الموقع.
